Conceptos

URL

URL son las siglas en inglés de “Uniform Resource Locator” y prácticamente es la dirección de cualquier elemento (página visible) en el internet.[1]

MIME

MIME son las siglas en ingles de “Multipurpose Internet Mail Extensions”, es un estándar que permitió expandir las capacidades del correo electrónico y también el adjuntar documentos (como imágenes, sonido y texto) en un mensaje.[2]

Métodos y códigos de errores en HTTP

[3][4] Errores de cliente:
400 Bad Request
El servidor no pudo interpretar la solicitud por errores de sintaxis.
401 Unauthorized
Es necesario autenticar para obtener la respuesta solicitada.
403 Forbidden
El cliente no posee los permisos necesarios para cierto contenido.
404 Not Found
El servidor no pudo encontrar el contenido solicitado.
408 Request Timeout
El servidor quiere desconectar esta conexión sin usar.

Errores de servidor:
500 Internal Server Error
El servidor ha encontrado una situación que no sabe cómo manejarla.
501 Not Implemented
El método solicitado no esta soportado por el servidor y no puede ser manejada.
502 Bad Gateway
Mientras el servidor trabaja como puerta de enlace para obtener una respuesta necesaria para manejar la petición, obtuvo una respuesta inválida.
503 Service Unavailable
El servidor no está listo para manejar la petición, esto comúnmente pasa porque está caído por mantenimiento o está sobrecargado
504 Gateway Timeout
El servidor está actuando como una puerta de enlace y no puede obtener una respuesta a tiempo.
505 HTTP Version Not Supported
La versión de HTTP usada en la petición no está soportada por el servidor.

Metodos:
GET
Este método solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos.
HEAD
El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
POST
El método POST se utiliza para enviar una entidad a un recurso en específico.
PUT
El modo PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
DELETE
El método DELETE borra un recurso en específico.
CONNECT
El método CONNECT establece un túnel hacia el servidor identificado por el recurso.
OPTIONS
El método OPTIONS es utilizado para describir las opciones de comunicación para el recurso de destino.
TRACE
El método TRACE realiza una prueba de la ruta completa que sigue el mensaje.
PATCH
El método PATCH es utilizado para aplicar modificaciones parciales a un recurso.

XML

XML son las siglas en ingles de “Extensible Markup Language”, este lenguaje fue diseñado para almacenar y transportar datos. En XML el autor debe crear sus propias etiquetas a diferencia de HTML.[5]

XHTML

XHTML es el “HTML extensible”, es practiamente HTML definido como XML, en lo que cambia con HTML es que todo debe estar bien escrito y bien anidado, esto con el fin de lograr que la aplicación corra adecuadamente en cualquier dispositivo.[6]

HTML5

HTML5 es la última versión de HTML, la cual incorpora nuevas tecnologías para aumentar la diversidad y el alcance de las páginas web.[7]

JS

JavaScript es un lenguaje multiparadigma ampliamente utilizado en el desarrollo de paginas web, dentro de esta rama su función es la de dictar el comportamiento de las paginas, además también se usa en otros programas como mongoDB y couchDB.[8]

PHP

PHP es el acrónimo de “Hypertext Preprocessor”, sirve para crear paginas web dinámicas e interactivas, lo que distingue a PHP de JS es que JS trabaja del lado del cliente y PHP del lado del servidor, donde genera HTML y se lo manda al cliente.[9]

JSP

“Java Server Pages” es una tecnologia similar a PHP y JS en el sentido en el que permite crear paginas web dinamicas e interactivas, además tiene acceso a todas las API’s de Java.[10]

AJAX

AJAX es el acrónimo de “Asynchronous Javascript and XML”, en corto AJAX permite que un servidor y un navegador intercambien información (XML, JSON, HTML) de forma asíncrona mediante programas escritos en Javascript.[11]
